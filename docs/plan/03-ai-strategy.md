## AI 核心机制

```
用户问题 + 用户背景
        ↓
规则引擎输出（结构化 JSON）──→ AI Prompt（护栏）
        ↓
  LLM 生成回复（自然语言）
```

### 数据流

1. **输入**: 规则引擎 JSON + 用户背景 + 具体问题
2. **Prompt**: 专业提示词模板（CoT、Few-shot、经典引用约束）
3. **输出控制**: 规则 JSON 作为事实护栏，禁止 AI 编造未验证内容

### 约束红线

| 规则 | 说明 |
|------|------|
| **护栏机制** | AI 回复必须基于规则引擎 JSON，不得脱离数据自由发挥 |
| **禁止编造** | 不得编造不存在的神煞、格局、五行关系 |
| **免责声明** | 每次回复末尾附带免责："非科学预测，仅供娱乐参考" |
| **敏感过滤** | 拒绝医疗诊断、法律建议、金融投资等绝对化建议 |

## AI 功能规划

| 功能 | 说明 | Phase |
|------|------|:---:|
| **智能对话助手** | 多轮对话，记忆上下文，理解命理问题 | Phase 2 |
| **个性化报告生成** | 一键生成，支持导出 PDF / 长图 | Phase 2 |

## 提示词工程（Prompt Engineering）

### 模板结构

```
System: 你是专业的八字命理分析师，基于提供的排盘数据进行解读。
        不得编造数据之外的内容。每次回复结尾附加免责声明。

Context: { 规则引擎 JSON 输出 }

User: { 用户问题 }

Rules:
- 必须引用 JSON 中的具体数据
- 禁止使用绝对化断言（"一定""必然"）
- 用平和、客观的语气
- 结尾: "以上内容仅供参考，祝您生活愉快。"
```

## 集成架构

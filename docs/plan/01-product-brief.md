## 项目定位

| 维度 | 内容 |
|------|------|
| **项目名称** | 八字排盘 & AI 智能命理助手（Bazi AI / 八字智解） |
| **产品定位** | 精准、专业、好看的新中式 AI 八字命理工具 |
| **核心理念** | 传统子平法为核心，现代 AI 技术为辅助 — **计算准确 + 规则透明 + 文化尊重** |
| **目标用户** | 25-45 岁都市年轻群体（事业/婚姻/人生焦虑者）、传统文化爱好者 |

## 核心价值

| 维度 | 说明 |
|------|------|
| **精确计算** | 基于 `lunar-typescript`，节气换月、起运天数精确计算 |
| **结构化规则引擎** | 每条批注可解释、可溯源、可验证 |
| **AI 自然语言** | 多轮对话 + 个性化互动，规则 JSON 作为事实护栏 |
| **新中式 UI** | 朱红 + 金色 + 深色背景，专业且美观 |

---

## 核心架构原则

```
┌──────────────────────────────────────────┐
│         外围功能层（可变、可增删）          │
│   AI 对话 · 报告导出 · UI 展示 · 分享      │
│              ↑ 只通过契约消费              │
│         contracts/engine-api.md           │  ← 接口契约（宪法）
│              ↑ 稳定 API                   │
│         核心排盘引擎（不可变）              │
│   calculator · pattern · annotation       │
└──────────────────────────────────────────┘
```

- **核心引擎封版后不修改**：已完成的八字计算、格局判断、批注流水线作为不可变内核
- **外围功能通过契约接入**：AI 对话、报告生成、UI 改版等，只能通过 `engine/index.ts` 公开接口消费
- **新增功能不碰引擎**：任何外围模块禁止直接引入 `engine/` 内部实现

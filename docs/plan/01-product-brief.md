| 维度 | 说明 |

| **古籍真传校准** | 核心规则经多部经典古籍逐条交叉校验，每条决策可溯源至原著 |

## 核心架构原则

```
┌──────────────────────────────────────────────────┐
│           外围功能层（可变、可增删）                 │
│   AI 对话 · 报告导出 · UI 展示 · 管理后台 · 数据库   │
│              ↑ 只通过契约消费                       │
│         contracts/engine-api.md                   │  ← 接口契约（宪法）
│              ↑ 稳定 API                             │
├──────────────────────────────────────────────────┤
│         核心推演引擎 engine/（不可变内核 · 算法黑盒） │
│   calculator · pattern · annotation                │
│   所有核心决策规则封版，只修 bug 不加功能            │
└──────────────────────────────────────────────────┘
```

- **核心引擎封版后不修改**：算法决策逻辑作为不可变内核，保持黑盒化
- **外围功能通过契约接入**：AI 对话、管理后台、数据库持久化等，只能通过 `engine/index.ts` 公开接口消费
- **新增功能不碰引擎**：任何外围模块禁止直接引入 `engine/` 内部实现
- **动态配置双轨路由**：管理后台数据库配置优先，自动回退 `.env` 静态文件（Phase 4+）

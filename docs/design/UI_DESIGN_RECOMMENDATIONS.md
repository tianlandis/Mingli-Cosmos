# 八字排盘 · 界面设计建议

> 基于 frontend-design 方法论，以「八字」本体世界为素材的定向设计提案
> 日期：2026-06-17

---

## 一、设计评判

### 当前设计

**现状评分：6.5/10** — 功能完整、风格统一，但陷入了一个 AI 设计的典型模板。

| 维度 | 评价 |
|------|------|
| 配色 | 深棕黑 `#1a1410` + 琥珀金 `amber-400` + 红褐渐变 — **典型的「深色底+单一亮色」AI 模板** |
| 质感 | 毛玻璃卡片 `backdrop-blur` — 与命理主题无关，这是 SaaS dashboard 的惯用手法 |
| 字体 | 微软雅黑 — 中性无衬线体，缺少文化辨识度 |
| 排版 | 标准的卡片-标题分区 — 适用于任何工具型 App，没有"八字"该有的性格 |
| 图标 | Emoji ☯ 🧠 ⚠️ — 缺乏质感，与暗色中式风格不匹配 |

**核心问题：把八字当成一个普通 SPA 来设计，而不是当成一件"命书"来设计。**

---

## 二、主题锚点

> 八字排盘 = 把出生时刻翻译成一幅命理图谱。它的本体质感来自：

- **宣纸** — 象牙白底，微温，纤维纹理
- **墨** — 从炭黑到淡灰，湿笔与干笔
- **朱砂印泥** — 印章的红色，是唯一的强色
- **手写命书** — 竖排小楷，红黑双色批注，圈点划线
- **罗盘/司南** — 圆形、刻度、定向

**设计方向：不是做传统网页，是做一本数字命书。**

## 三、Token 系统

### 3.1 调色板

```
宣纸底    #FBF7F0  — 页面主背景
淡墨栏    #EDE8DF  — 分区/卡片背景
墨色      #1C1914  — 正文标题
淡墨      #6B6459  — 辅助/说明文字
朱砂      #B83A2E  — 日主印章、关键强调、CTA 按钮
金箔      #B8973E  — 极少量的点缀（例：吉位标注）  
冷墨      #4A5568  — 弱/忌神标记（用于"不足"方向的视觉编码）
```

### 3.2 字体

```
展示：Noto Serif SC（思源宋体）— 标题、日主天干、干支大字
正排：Noto Sans SC（思源黑体）— 正文、表格、数据
数字：Tabular Nums feature — 表格中数字等宽对齐
```

**为什么不用楷体/行书展示字？** 可读性。命书上小楷很美，在 72dpi 屏幕上则发虚。思源宋体有古典骨架 + 屏幕优化，折中方案。

### 3.3 布局

**核心原则：竖轴流式 + 留白呼吸**

- 单列滚动（如展开的卷轴），不做多列 dashboard
- 每个区块之间用淡墨细线分隔（`border-t border-[#D8D2C8]`），**不用深色边框，不用 rounded 大圆角**
- 区块内部使用非对称留白：内容区偏左，右侧留白 20%（中国画的留白传统）
- 最大内容宽度 `max-w-3xl`（48rem，约 768px），比当前 `max-w-6xl` 收窄，更像书页
- 圆角全部降至 `rounded-sm`（2px）或直角，避免 SaaS card 的 `rounded-2xl` 质感

### 3.4 标志性元素（Signature）

**八字命盘作为视觉英雄，印成「印章」效果。**

当前设计把 BaziChart 作为结果列表里一张普通卡片。新方案：

- 命盘四柱 8 个字符（年干年支、月干月支、日干日支、时干时支）放在页面顶部居中
- 每柱上下排列（天干/地支），用大号思源宋体（`text-3xl`+）
- **日柱的日干（日主）用朱砂红印效果**：红色字符 + 外方框 + 轻微旋转 3-5°，模拟印章盖在宣纸上的质感
- 命盘下方署"出生时刻"信息（小字淡墨），如传统命书抬头

这是整个页面让人记住的地方 — **八字排盘的八字，就是 identity**。

## 四、分区重设计

### 英雄区 → 命盘抬头

```
┌──────────────────────────────────────┐
│                                      │
│         乙          戊          戊          丙      │
│         卯          申          辰          辰      │
│                                                      │
│                      · · ·                           │
│              公历 2000年8月14日 辰时                     │
│                   男命 · 顺行                          │
│                                                      │
└──────────────────────────────────────┘
```
- 四柱大字竖排，更像传统命书的写法（也可以横排以适应现代阅读习惯，但字号要大）
- 日主的「戊」用朱砂红 + 方形边框 + 3° 旋转
- 下方署信息用淡墨小字，冒号对齐

### 输入区 → 嵌入顶部

输入表单从"独立大卡片"收敛为页面顶部的紧凑工具栏。  
类似历代黄历的查找控件 — 快速、低调、工具感强。

```
┌──────────────────────────────────────────┐
│  [公历·农历] [♂·♀] [年▾] [月▾] [日▾] [时▾] [排盘→] │
└──────────────────────────────────────────┘
```
- 所有控件一行内联（桌面端），移动端折行
- `排盘` 按钮用朱砂红，其他控件用淡墨边框

### 内容区 → 命书章节

不再用独立卡片，而是连续的"命书章节"：

```
── 第一节 · 五行分布 ──────────────────────

  水平条形图，五行标签在左侧，条形向右延伸
  淡墨底色 + 对应五行色的填充条（见 3.1 调色板）

── 第二节 · 十神配置 ──────────────────────

  标签式排列，不再用表格
  每个十神一行：名称（墨色）+ 数量（淡墨）+ 位置（淡墨）
  日主的比肩/劫财用朱砂标记

── 第三节 · 日主强弱 ──────────────────────

  单行进度条，左"弱"右"强"
  刻度用墨色渐变
  下方列出各维度评分，每行一条淡墨横线

── 第四节 · 格局 ──────────────────────────

  格局名称用思源宋体大字（text-2xl），如题匾
  品质标记（上等/中等/普通）用小号朱砂印章

── 第五节 · 大运流年 ──────────────────────

  时间线改为竖向时间轴（不是横向滚动时间线）
  左侧年份，右侧干支 + 简短判词
  当前所在的十年用金箔细线高亮左侧

── 第六节 · 专题 ──────────────────────────

  六项专题（性格/事业/财运/婚姻/健康/子女）
  6 个水平标签切换（tab），一次展示一个，不再平铺 6 宫格
  标题用思源宋体，正文用思源黑体，判词重点用朱砂点

── 第七节 · 综合 ──────────────────────────

  三行以内总结
  建议用编号列表，每个编号用朱砂小圆点
```

## 五、交互与动效

### 5.1 页面加载序列

```
排盘按钮点击
    ↓
输入区收起（上滑 300ms）
命盘抬头从淡墨渐显（500ms fade-in + 从上向下 20px 滑动）
章节依次出现（staggered，每节延迟 150ms）
    ├─ 五行分布
    ├─ 十神配置
    ├─ 日主强弱
    ├─ 格局
    ├─ 大运流年
    ├─ 专题
    └─ 综合
```

### 5.2 微交互

| 元素 | 动效 |
|------|------|
| 排盘按钮 hover | 朱砂色加深至 `#9B2C22` + scale(1.02) |
| 章节标题 | scroll-triggered: 从淡墨变墨色的过渡 |
| 专题 tab 切换 | 内容区 `fade` 200ms + 指示条滑动 |
| 大运时间轴 | 当前项朱砂圆点脉冲（`ring` 动画） |

### 5.3 减少动效

尊重 `prefers-reduced-motion`，全部动画降级为无过渡。

## 六、移动端适配

| 断点 | 宽度 | 行为 |
|------|------|------|
| `sm` | < 640px | 输入表单纵向堆叠，命盘字号缩小至 `text-xl`，章节留白取消 |
| `md` | ≥ 640px | 输入表单一行显示，命盘 `text-2xl` |
| `lg` | ≥ 1024px | `max-w-3xl` 居中，右侧留白空间增大 |

## 七、实施优先级

### 🔴 P0 — 核心视觉重构（最大收益）

1. **调色板切换**：从深棕黑 → 宣纸底，这是方向性转变
2. **命盘英雄区**：四柱大字居中 + 日主印章效果
3. **章节化布局**：卡片 → 章节 + 淡墨分隔线
4. **字体替换**：微软雅黑 → 思源宋体 + 思源黑体

### 🟡 P1 — 品质提升

5. **输入区紧凑化**：独立卡片 → 内联工具栏
6. **大运竖轴**：横向时间线 → 竖向年表
7. **专题 tab 化**：6 宫格 → 切换 tabs
8. **加载序列**：staggered 章节入场动画

### 🟢 P2 — 细节打磨

9. **印章微调**：日主朱砂印的旋转 + 方框 + 纹理
10. **五行 bar 重设计**：水平条形替代柱状图
11. **响应式细化**：`sm:` / `md:` / `lg:` 三断点

## 八、风险提示

| 风险 | 影响 | 应对 |
|------|------|------|
| 浅色底在强光环境仍然可读性不佳 | 用户可能偏好暗色模式 | 保留暗色方案作为 `prefers-color-scheme: dark` 的备选 |
| 思源宋体中文字重加载慢 | FOUT 闪烁 | 使用 `font-display: swap` + 子集化 |
| 竖向大运时间轴占用垂直空间大 | 滚动距离增加 | 限制展开至 4 条，折叠查看更多 |
| 朱砂红在色弱用户眼中辨识度低 | 可访问性不足 | 日主印章同时使用"红 + 方形框"双重编码 |
| 整体偏离主流 SPA 视觉 | 用户可能认为"不够现代" | 八字是传统文化工具，古典感是价值而非缺陷 |

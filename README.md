# 八字排盘 · Bazi Paipan

> **正宗子平法八字命理排盘引擎** — 传统规则为骨，现代 AI 为翼

基于 **React 19 + TypeScript + Vite + Tailwind CSS 4** 构建的新中式八字命理 Web 应用。核心引擎严格遵循传统子平法理论，从节气换月、藏干深度、格局取用到大运起运，每一步计算均可溯源至经典典籍。

---

## 🎯 为什么这是"真传"

市面上大多数八字产品用简表查月、固定藏干、笼统格局，**这里不是**：

| 特性 | 本项目 | 市面常见 |
|------|:---:|:---:|
| **节气换月** | ✅ 精确到节气交节时刻 | ❌ 固定日期换月 |
| **地支藏干** | ✅ 完整本气/中气/余气 | ❌ 仅取本气或简化 |
| **月令分金** | ✅ 藏干天数刻度尺精确到天 | ❌ 不支持 |
| **格局判断** | ✅ V2.0 四正/透干/分金三路径 + 7 组合 + 4 破格 | ❌ 简单正格枚举 |
| **起运天数** | ✅ 分金依赖精确计算 | ❌ 粗略估算 |
| **十神推导** | ✅ 以日主为中心完整推导 | ❌ 部分缺失 |
| **大运排法** | ✅ 阴阳顺逆 + 节气换运 | ❌ 简化排法 |
| **神煞** | ✅ 8 种经典神煞 + MBTI 人格映射 | ❌ 缺失或不准确 |

> **核心理念**：计算准确 + 规则透明 + 文化尊重。每条批注可解释、可溯源、可验证。

## 📐 核心算法：13 步批注流水线

```
用户输入（公历/农历 + 性别）
        ↓
① computeBazi()                排盘计算
   ├─ 四柱八字（年柱·月柱·日柱·时柱）
   ├─ 藏干深度（本气·中气·余气）
   ├─ 六十甲子纳音
   ├─ 十神推导（以日主为中心）
   └─ 大运列表（起运天数 + 十年一运）
        ↓
② generateAnnotation()         批注流水线（严格有序）
   ├─ Step  1: 日主强弱评分    得令/得地/得势/通根/坐支
   ├─ Step  2: 用神忌神取用    扶抑/从强/调候 三策略
   ├─ Step  3: V2.0 格局判断   四正取格→透干取用→月令分金
   │   ├─ 3.1 四正月直接取格
   │   ├─ 3.2 透干优先级（月干＞年干＞时干）
   │   ├─ 3.3 月令分金（藏干天数刻度尺）
   │   ├─ 3.4 格局组合判定（7 种）
   │   ├─ 3.5 破格风险检测（4 类）
   │   └─ 3.6 MBTI 人格映射
   ├─ Step  4: 五行平衡分析
   ├─ Step  5: 十神概况统计
   ├─ Step  6: 大运逐运解读
   ├─ Step  7: 当前流年分析
   ├─ Step  8: 人生关键节点
   ├─ Step  9: 地支关系分析    刑冲破害合三合三会空亡
   ├─ Step 10: 神煞分析        天乙/文昌/桃花/驿马/华盖/金舆/禄/羊刃
   ├─ Step 11: 六大专题批注    性格·事业·财运·婚姻·健康·子女
   ├─ Step 12: 综合命局总览
   └─ Step 13: 改运建议
        ↓
   AnnotationResult（完整结构化输出）
```

## 🏗️ 核心架构：引擎封版 + 外围插件化

```
┌──────────────────────────────────────┐
│    外围功能（可变、可增删）            │
│    AI 对话 · 报告导出 · 分享海报       │
│         ↑ 只通过契约消费              │
│    contracts/engine-api.md            │  ← 接口"宪法"
│         ↑ engine/index.ts             │
├──────────────────────────────────────┤
│    核心引擎 engine/（不可变内核）      │
│    calculator · pattern · annotation  │
│    已完成封版，只修 bug 不加功能       │
└──────────────────────────────────────┘
```

> **红线**：任何外围模块禁止直接引用 `engine/` 内部实现，只能通过 `engine/index.ts` 的公开接口消费。确保以后加 AI、改 UI、导出 PDF，核心算法纹丝不动。

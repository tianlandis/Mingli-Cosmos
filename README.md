# 八字排盘 · Bazi Paipan

> **正宗子平法八字命理排盘引擎** — 传统规则为骨，现代 AI 为翼

基于 **React 19 + TypeScript + Vite + Tailwind CSS 4** 构建的新中式八字命理 Web 应用。核心引擎以 **Python MCP（tengods）为算法权威参考**，严格遵循传统子平法理论，从节气换月、藏干深度、格局取用到大运起运，每一步计算均可溯源至经典典籍。

---

## 🎯 为什么这是"真传"

市面上大多数八字产品用简表查月、固定藏干、笼统格局，**这里不是**：

| 特性 | 本项目 | 市面常见 |
|------|:---:|:---:|
| **节气换月** | ✅ 精确到节气交节时刻 | ❌ 固定日期换月 |
| **地支藏干** | ✅ 完整本气/中气/余气（Python MCP 对齐） | ❌ 仅取本气或简化 |
| **月令分金** | ✅ 藏干天数刻度尺精确到天（动态高精度校准） | ❌ 不支持 |
| **格局判断** | ✅ 四正/透干/分金三路径（多维联合防偏校验） | ❌ 简单正格枚举 |
| **起运天数** | ✅ 分金依赖精确计算 | ❌ 粗略估算 |
| **十神推导** | ✅ 以日主为中心完整推导 | ❌ 部分缺失 |
| **大运排法** | ✅ 阴阳顺逆 + 节气换运 | ❌ 简化排法 |
| **神   煞** | ✅ 8 种经典神煞 + MBTI 人格映射 | ❌ 缺失或不准确 |

> **核心理念**：计算准确 + 规则透明 + 文化尊重。每条批注可解释、可溯源、可验证。

### 🔬 算法权威基准

我们的引擎不依赖“黑盒”逻辑，所有推演均基于 Python MCP (tengods_bazi_mcp) 定义的严苛标准进行对齐。本系统在以下核心命理维度实现了与权威算法的逻辑一致性：

格局取用标准：采用四正取格与透干优先级多维校验，有效规避了比劫干扰，确保格局判定精准。

分金迭代逻辑：内嵌标准分金算法，精确支持正反向迭代逻辑，确保藏干深度的精确计算。

神煞与映射：基于标准命理规则与现代人格映射（MBTI）的双重验证，确保推演既符合古法，又具备现代解读价值。

欲了解详细的算法对齐细节与推演规则，请参阅：docs/arch/ALGORITHM-AUTHORITY.md

## 📐 核心算法：13 步批注流水线

我们的引擎采用模块化递进处理机制，确保从生辰数据到最终建议的每一个推演环节均逻辑闭环：

| 阶段 | 核心模块 | 描述 |

| | 02. 推演初始化 | 十神体系构建及日主强弱多维评分 |
| **格局推演** | 03. 核心取格引擎 | **核心算法封版模块**：基于四正、透干优先级及分金迭代的高精度定格 |
| | 04. 五行平衡分析 | 全局五行动态平衡扫描与缺失补救方案 |
| | 05. 十神概况统计 | 基于权重的命局能量强度分布统计 |
| **深度解读** | 06. 大运逐运推演 | 结合大运干支与命局的十年动态关联分析 |
| | 07. 流年趋势捕捉 | 当前周期内关键动态影响与趋势预测 |
| | 08. 人生关键节点 | 结合运势波动的重大机遇与挑战阈值判定 |
| **多维度分析** | 09. 地支关系校验 | 刑冲破害合三合三会等复杂关系的解析 |
| | 10. 神煞星盘映射 | 经典神煞与命局关系的自动校验与匹配 |
| | 11. 专题精细化批注 | 性格、事业、财运、婚姻、健康、子女六大维度的AI语义增强 |
| **综合输出** | 12. 命局总览决策 | 综合权重的多维度归纳总结 |
| | 13. 改运优化建议 | 策略性调整建议与行事趋避指南 |

> **提示**：核心流水线已实现“黑盒化”封装，外围模块仅通过标准化接口调用，确保了算法实现层面的绝对独立与不可篡改。

## 🏗️ 核心架构：引擎封版 + 外围插件化

```
┌──────────────────────────────────────┐
│    外围功能（可变、可增删）            │
│    AI 对话 · 报告导出 · 分享海报       │
│         ↑ 只通过契约消费              │
│    contracts/engine-api.md            │  ← 接口"宪法"
│         ↑ engine/index.ts             │
├──────────────────────────────────────┤
│    核心引擎 engine/（不可变内核）      │
│    calculator · pattern · annotation  │
│    已完成封版，只修 bug 不加功能       │
└──────────────────────────────────────┘
```

> **红线**：任何外围模块禁止直接引用 `engine/` 内部实现，只能通过 `engine/index.ts` 的公开接口消费。确保以后加 AI、改 UI、导出 PDF，核心算法纹丝不动。

## 🚀 快速开始

本项目已实现前后端同构（Vite + Hono），统一端口全栈部署。

```bash
# 安装依赖
npm install

# 启动开发服务器 (前端热更新 + API 服务)
# 默认地址：http://localhost:3001
npm run dev

# 生产环境构建 (静态打包)
npm run build

# 启动生产服务器 (单端口 Serve Static)
npm run start
```
